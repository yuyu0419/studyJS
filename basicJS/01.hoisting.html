Hoisting

console창에서도 확인 가능

var의 hoisting의 예
console.log(a) //undefined
a = 0;
console.log(a) //0
var a = 1;


호이스팅은 선언부만 올라가므로 a는 1이 되지 않는다.
var의 이런 문제때문에 es6부터 let이 도입되었다.
let은 위와 같은 상황에서는 Uncaught ReferenceError가 발생한다.
만약 이 상황에서 a를 1로 만들고 싶다면 a에 값을 주지 않는다.


let과 var의 비교
let b; //var b도 가능함
console.log(b) //undefinde
b = 0;
console.log(b) //0
b = 1;
console.log(b) //1
